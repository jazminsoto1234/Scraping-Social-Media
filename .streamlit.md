# Streamlit Dashboard - Implementation Prompt

## Overview

Build a professional **Streamlit-based Comment Analytics Dashboard** for analyzing social media comments (TikTok + Facebook) that have been scraped and classified using the existing ML pipeline.

The dashboard provides a clean, enterprise-grade interface to explore comment sentiment, themes, and patterns across multiple videos.

## Data Pipeline

### Loading & Processing
- Load `comments.json` based on platform selection (TikTok or Facebook)
- Group comments by `video_id` to identify unique videos
- Extract video keyword/topic from comments:
  - Use most common meaningful nouns/phrases in comments
  - Format examples: "Starlink Launch", "5G Coverage", "New Plans"
  - Store as metadata for video navigator labels
- Cache all data with `@st.cache_data` for performance

**Color Palette** (consistent throughout):
- **Positivo**: `#22C55E` (Vibrant Green)

- **Cards**: `#FFFFFF` (White)

### Analytics Section (2-Column Layout)

**LEFT COLUMN: Video Selection & Classification**

1. **Video Navigator** (Horizontal Scrollable Cards)
   - Clickable card selector for each video
   - Layout: `[ Video 1 - Starlink Launch (150) ]  [ Video 2 - 5G Coverage (98) ] ...`
   - Selected card: Blue border + light blue background
   - Unselected cards: Light gray background
   - Scrollable for 20+ videos
   - Store selection in `st.session_state['selected_video_id']`

2. **Video Info & Breakdown** (Below Navigator)
   ```
   Video Keyword/Topic (large, bold)
   Video ID: 7632494760351091986
   Total Comments: 150
   
   Classification Breakdown:
   ║████████████║ Positivo: 89 (59.1%)
   ║██████║ Negativo: 35 (23.3%)
   ║███║ Informativo: 15 (10%)
   ║██║ Neutral: 11 (7.3%)
   ```
   - **Chart**: Horizontal bars using Plotly (minimal design)
   - **Bar Style**: Smooth rounded edges (no sharp vertices) - use `cornerradius` or `shape` styling for soft borders
   - **Colors**: Match classification palette for each bar
   - **Labels**: Count + percentage inside bars (white text, centered)
   - **Hover Behavior**: Interactive tooltips showing classification name, count, and percentage
   - **Sorting**: Display bars in descending order (highest count first)
   - **Visual Hierarchy**: Add separator line and "Top Classification" badge below chart
   - **No axes, gridlines, or toolbar** — clean, minimal appearance
   - **Implementation**: `st.plotly_chart()` with `use_container_width=True` and `config={"displayModeBar": False}`

**RIGHT COLUMN: Word Clouds**

Two side-by-side word clouds (filtered to selected video only):

1. **Positivo Word Cloud**
   - Color: `#22C55E` (Green)
   - Content: Top 30-40 most common words from positive comments
   - Display size: figsize=(10, 5)
   - Implementation: Use `wordcloud` library + `st.pyplot()`

2. **Negativo Word Cloud**
   - Color: `#DC2626` (Red)
   - Content: Top 30-40 most common words from negative comments
   - Display size: figsize=(10, 5)
   - Implementation: Use `wordcloud` library + `st.pyplot()`

## Implementation Architecture

### Session State
```python
st.session_state:
  - selected_video_id: str         # Currently selected video
  - current_page: int              # Pagination page number
  - selected_classifications: list # Active filter [positivo, negativo, ...]
  - search_query: str              # Search term in comments
```

### Caching Strategy
```python
@st.cache_data
def load_comments(platform: str) -> list[dict]:
    """Load JSON comments (cached)"""

@st.cache_data
def group_by_video(comments: list[dict]) -> dict:
    """Group comments by video_id (cached)"""

@st.cache_data
def extract_keywords(comments: list[dict]) -> str:
    """Extract video keywords from comments (cached)"""

@st.cache_data
def generate_wordcloud(texts: list[str], color: str):
    """Generate word cloud figure (cached)"""
```

### Performance Optimizations
- Load all comments once and cache
- Generate word clouds only when video selected
- Reuse cached word clouds on video re-selection
- Debounce search input (small delay)
- Lazy-load components: render only visible section
